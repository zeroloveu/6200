# `scripts/fastapi-chain.js` 模块说明

## 1. 模块定位

`scripts/fastapi-chain.js` 是项目中的 Node.js 链桥脚本。  
它负责真正使用 Ethers.js 与区块链节点交互，是 Python 后端调用链上功能的执行端。

## 2. 支持的命令

该脚本目前支持三类命令：

- `deploy`：部署新的投票合约
- `summary`：读取合约摘要信息
- `voter-action`：查询某个地址最近一次链上投票动作

## 3. `deploy` 部分

### 3.1 主要流程

部署流程大致如下：

1. 读取合约编译产物
2. 创建 `JsonRpcProvider`
3. 使用部署私钥构造钱包对象
4. 生成部署交易
5. 获取 `nonce`、手续费信息和 gas 估算
6. 对交易进行签名
7. 广播交易
8. 等待交易回执
9. 返回合约地址、交易哈希、区块高度等信息

### 3.2 稳定性增强

该脚本已经针对真实部署场景做过增强，包括：

- `already known` 错误恢复
- 请求超时恢复
- 回执轮询恢复
- 在交易已广播但尚未确认时返回 `PENDING` 状态

这样做可以降低“链上已发出交易，但网页误判为失败”的概率。

## 4. `summary` 部分

该命令会读取合约的实时状态，包括：

- 选举标题
- 合约拥有者
- 开始时间与结束时间
- 投票状态
- 总票数
- 弃权数
- 已注册选民数
- 候选项列表及其票数
- 当前查看者是否在白名单中、是否已投票
- 若投票结束，则读取赢家信息

## 5. `voter-action` 部分

该命令会查询指定钱包地址的事件记录，包括：

- `VoteCast`
- `Abstained`

然后按照区块号和日志顺序合并排序，返回最近一次有效动作。  
该结果会被 Python 后端用于校验用户提交的同步信息是否真实。

## 6. 设计价值

这个脚本的意义在于把复杂的链上交互集中到一个地方处理。  
相较于在 Python 中直接嵌入 Web3 逻辑，这种设计有几个优点：

- 复用 Ethers.js 的生态与兼容性
- 保持 FastAPI 代码更清晰
- 更容易定位链上交互错误
- 部署、查询、事件扫描逻辑都能集中维护
