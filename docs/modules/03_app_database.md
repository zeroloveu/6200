# `app/database.py` 模块说明

## 1. 模块定位

`app/database.py` 是项目的数据库基础设施模块。  
它负责环境变量加载、数据库连接建立、会话创建、表初始化以及兼容旧数据库结构的迁移处理。

## 2. 主要职责

- 从 `.env` 文件中读取配置
- 根据 `APP_DATABASE_URL` 创建 SQLAlchemy 引擎
- 提供数据库会话工厂
- 提供 `get_db()` 依赖注入给 FastAPI 路由使用
- 在应用启动时初始化数据库表
- 自动补齐旧版本数据库缺失的字段

## 3. 核心函数

### 3.1 `load_local_env()`

该函数会读取项目根目录下的 `.env` 文件，并将内容写入当前进程环境变量。  
项目当前实现采取“以 `.env` 为准”的策略，避免终端残留环境变量覆盖项目配置。

### 3.2 `get_db()`

该函数是 FastAPI 中的数据库依赖入口。  
每次请求会获取一个新的数据库会话，请求结束后自动关闭，避免连接泄漏。

### 3.3 `init_db()`

该函数会在应用启动时执行，完成两件事：

- 创建当前 ORM 模型对应的数据表
- 触发兼容旧表结构的补字段逻辑

### 3.4 `ensure_legacy_columns()`

这是一个轻量级的“兼容迁移”函数。  
它会检查旧数据库中是否缺少以下字段：

- 用户表中的 `wallet_address`
- 投票表中的链上相关字段
- 投票记录表中的 `chain_tx_hash`

如果缺少，就自动执行 `ALTER TABLE` 补齐。

## 4. 设计思路

### 4.1 适合课程项目的轻量迁移方式

本项目没有引入 Alembic 之类的正式迁移框架，而是采用应用启动时自动检测并补字段的方式。  
这种方案的优点是：

- 上手快
- 依赖少
- 适合单机演示与课程答辩

缺点是：

- 复杂迁移能力有限
- 不适合大规模生产环境

### 4.2 SQLite 优先兼容

项目默认使用 SQLite，因此在创建引擎时会加上 `check_same_thread=False`，以适应 FastAPI 请求处理环境。

## 5. 在系统中的作用

如果说 `models.py` 定义了“数据长什么样”，那么 `database.py` 负责“这些数据如何真正落到数据库中并被请求安全地访问”。  
它属于项目中非常基础但又非常关键的底层模块。
