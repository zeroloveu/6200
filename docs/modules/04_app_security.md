# `app/security.py` 模块说明

## 1. 模块定位

`app/security.py` 是系统的密码安全模块。  
它负责用户密码的安全存储和登录验证，避免明文密码直接写入数据库。

## 2. 主要功能

- `hash_password(password)`：对用户密码进行加盐哈希
- `verify_password(password, stored_hash)`：校验输入密码是否与数据库中的哈希值匹配

## 3. 核心实现

### 3.1 使用 `scrypt`

模块使用 Python 标准库中的 `hashlib.scrypt()` 进行密码派生。  
`scrypt` 适合密码场景，原因在于：

- 计算成本较高
- 具备一定抗暴力破解能力
- 支持可配置参数

### 3.2 随机盐值

每次生成密码哈希时，系统都会通过 `secrets.token_bytes(16)` 生成随机盐值。  
这样即使两个用户设置相同密码，数据库中的最终哈希结果也不会相同。

### 3.3 安全比较

`verify_password()` 使用 `hmac.compare_digest()` 做常量时间比较，减少传统字符串比较可能带来的侧信道风险。

## 4. 存储格式

系统最终把密码存储成如下格式：

`scrypt$n$r$p$salt$derived_key`

这种格式的优点是：

- 包含算法名
- 包含参数
- 包含盐值
- 包含最终派生结果

后续校验时可以直接从字符串中恢复配置，不需要额外数据库字段。

## 5. 在系统中的价值

虽然这是一个规模不大的课程项目，但用户认证依然是系统可信性的基础。  
`app/security.py` 提供了最核心的一层口令保护，确保平台账号体系至少具备基本安全性。
