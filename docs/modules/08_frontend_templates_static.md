# 前端模板与静态资源模块说明

## 1. 模块定位

前端界面主要由以下两部分组成：

- `app/templates/`：Jinja2 页面模板
- `app/static/`：样式、脚本库和静态资源

该模块负责把后端数据转换为可交互的网页界面。

## 2. 模板层结构

### 2.1 `base.html`

基础模板，负责定义全站公共布局，例如：

- 顶部导航
- 全局消息提示
- 页面基础结构
- 公共字体与样式入口

其他页面模板通常都基于这个模板扩展。

### 2.2 页面模板

- `index.html`：首页，展示项目介绍、公开结果和可访问投票
- `dashboard.html`：用户工作台
- `login.html` / `register.html`：认证页面
- `profile.html`：钱包绑定页面
- `poll_form.html`：创建或编辑投票页面
- `poll_detail.html`：进行中的投票详情页
- `poll_results.html`：已结束投票的结果页

## 3. 样式层

### 3.1 `styles.css`

这是项目的主样式文件，负责：

- 全局排版
- 颜色系统
- 按钮与表单规范
- 卡片布局
- 详情页和结果页样式
- 响应式适配

当前版本已经做过一次现代化升级，特点包括：

- 字体更大
- 信息层级更清晰
- 风格更现代、更标准
- 兼顾桌面端和移动端

### 3.2 `vendor/ethers.umd.min.js`

浏览器端的 Ethers.js 依赖，用于在前端发起钱包签名与链上交易。

## 4. 前后端协作方式

前端模板并不是纯静态页面，它会接收后端动态注入的数据，例如：

- 当前用户信息
- 投票状态
- 链上合约地址
- 候选项与票数
- 允许用户列表
- 错误消息与成功消息

因此前端模板层本质上是“展示层 + 轻量交互层”。

## 5. 在系统中的作用

如果智能合约负责“可信”，后端负责“业务”，那么前端模块负责“可用性”。  
它决定了用户是否能够顺畅地完成注册、绑定钱包、创建投票、查看结果和发起投票等全过程。
