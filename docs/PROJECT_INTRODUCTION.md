# Secure Voting 混合版项目说明

## 1. 项目概述

本项目现在采用“前端/后端管理 + 链上投票记录”的混合架构：

- FastAPI 负责用户注册登录、会话管理、投票创建与编辑、允许投票名单管理、结果页面展示
- Solidity 智能合约负责真正的投票写链、计票和结果公开
- 浏览器钱包负责让被允许的用户亲自发起链上投票交易

这意味着：

- 应用层负责“管理流程”
- 区块链负责“结果真源”

相较于早期仅依赖命令行脚本部署和交互的方案，当前版本已经统一为 Web 应用主导的使用方式，不再依赖旧的独立脚本流程。

## 2. 当前系统能做什么

### 2.1 账号系统

- 用户可以注册和登录
- 密码会加密存储
- 每个账号可以绑定一个钱包地址

### 2.2 投票管理

- 任意注册用户都可以创建投票
- 创建投票时可以设置：
  - 投票议题
  - 开始时间
  - 结束时间
  - 候选项
  - 允许投票的用户名单
- 创建投票时，后端会自动部署一份独立的 `SecureVoting` 合约
- 创建者可以在无人投票前修改或删除投票

### 2.3 链上投票

- 只有在允许名单中的用户才能参与投票
- 被允许的用户必须先在系统中绑定钱包地址
- 用户在投票详情页通过浏览器钱包发起链上交易
- 用户可以：
  - 投给某个候选项
  - 选择弃权
- 每个钱包地址只能投票或弃权一次

### 2.4 公开结果

- 投票结束后，所有人都可以查看结果
- 结果页面读取的是链上合约状态，而不是本地数据库手工统计
- 页面会展示：
  - 各候选项得票数
  - 弃权数
  - 允许投票总人数
  - 胜者或平票状态
  - 合约地址与区块浏览器链接

## 3. 技术架构

### 3.1 Web 层

- FastAPI
- SQLAlchemy
- Jinja2 模板
- Session 登录
- SQLite 默认存储

### 3.2 链上层

- Solidity
- Hardhat
- Ethers.js

### 3.3 混合桥接层

项目中保留了一份新的桥接脚本：

- `scripts/fastapi-chain.js`

它只服务于新的 FastAPI 混合架构，负责：

- 部署投票合约
- 读取合约摘要
- 查询某个钱包是否已经在链上投票或弃权

## 4. 数据流说明

系统的标准流程如下：

1. 用户在 Web 页面注册账号
2. 用户在个人资料页绑定钱包地址
3. 创建者发布一个投票
4. 后端自动把该投票部署为链上合约
5. 被允许的用户打开投票详情页
6. 浏览器钱包弹出签名确认
7. 用户提交链上交易完成投票或弃权
8. 网站同步交易哈希并展示结果
9. 投票结束后，全体用户都能查看链上结果

## 5. 合约规则

当前 `SecureVoting` 合约支持：

- 白名单选民
- 投票时间窗口限制
- 每个地址只能参与一次
- 正常投票
- 链上弃权
- 结束后查询所有结果
- 查询赢家或平票状态

## 6. 当前环境变量

当前项目只保留新方案所需的主要环境变量：

- `APP_SECRET_KEY`
- `APP_DATABASE_URL`
- `APP_TIMEZONE`
- `APP_CHAIN_RPC_URL`
- `APP_CHAIN_PRIVATE_KEY`
- `APP_CHAIN_NETWORK_NAME`
- `REPORT_GAS`

其中：

- `APP_SECRET_KEY` 用于 FastAPI 会话签名
- `APP_DATABASE_URL` 用于应用数据库连接
- `APP_CHAIN_RPC_URL` 用于连接链上节点
- `APP_CHAIN_PRIVATE_KEY` 用于后端部署投票合约
- `APP_CHAIN_NETWORK_NAME` 当前建议使用 `sepolia`

## 7. 当前已移除的旧方案

以下旧的独立命令行交互方案已经从项目中删除：

- 独立部署脚本
- 独立状态查询脚本
- 独立注册选民脚本
- 独立投票脚本

现在项目默认使用：

- FastAPI 页面管理投票
- 浏览器钱包完成链上投票
- FastAPI 页面读取链上结果

## 8. 适用场景

本项目适合：

- 课程项目
- 毕业设计原型
- 区块链投票系统演示
- Web 管理 + 链上可信结果的教学展示

本项目不适合：

- 正式政治选举
- 高隐私要求投票
- 大规模生产级治理系统

## 9. 总结

当前版本已经不再是“合约 + 若干命令行脚本”的演示项目，而是一套完整的混合式投票应用：

- 管理操作在 Web 端完成
- 投票记录写入区块链
- 结果由链上公开验证

这也是本项目目前最推荐、也是唯一保留的使用方式。
